Why?

Why was DummyJS created?

The truth is, using some of the existing functional testing tools made me feel stupid and unproductive. That's because they are not functional testing tools, but browser automation tools that you can program to perform functional tests. Naturally, the flexibility of being programmable comes with a price of complexity. I didn't need the flexibility, so I didn't want to pay the price.

DummyJS was created because I wanted a true functional testing tool that I could give a list of simple instructions, not unlike the instructions you would give a human tester.

Why should I use DummyJS?

If you want to write and run simple functional tests, performing basic actions and tests on webpages, you should at least give it a try.

Why shouldn't I use DummyJS?

If you want to do anything more complex than performing a list of actions and tests in sequence, or if you like writing tests in a real programming language, don't use DummyJS. DummyJS doesn't have functions, conditionals or loops.
